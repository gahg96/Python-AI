# 监管报送模块设计方案

> **金融机构合规经营的重要环节：及时、准确、完整地向监管部门报送各类报表和报告**

---

## 一、设计背景

### 1.1 监管报送的重要性

监管报送是金融机构与监管部门沟通的重要桥梁，是合规经营的重要组成部分：

- **法律义务**：根据《商业银行法》、《银行业监督管理法》等法律法规，金融机构有义务向监管部门报送各类报表
- **监管要求**：监管部门通过报表了解金融机构的经营状况、风险水平、合规情况
- **风险预警**：监管部门通过报表数据识别潜在风险，及时采取监管措施
- **合规证明**：按时、准确报送是金融机构合规经营的重要证明

### 1.2 监管报送的挑战

金融机构在监管报送过程中面临诸多挑战：

- **报表种类繁多**：需要报送的报表类型多达几十种，每种报表都有不同的格式和要求
- **报送频率高**：日报、周报、月报、季报、年报等不同频率的报表
- **数据要求严格**：数据必须准确、完整、及时，一旦出错可能面临监管处罚
- **格式要求复杂**：不同报表有不同的格式要求，需要严格按照监管要求填写
- **时间要求紧迫**：很多报表需要在短时间内完成，时间压力大
- **审核流程复杂**：报表需要经过多级审核，流程复杂

## 二、监管报送业务场景

### 2.1 日常监管报表报送

#### 场景一：资本充足率报表（1104报表）

**业务背景**：
- 报送频率：**季度报送**
- 报送时间：每季度结束后15个工作日内
- 监管部门：银保监会
- 报表编号：G01-G04系列

**报表内容**：
1. **G01报表**：资本充足率计算表
   - 核心一级资本构成
   - 其他一级资本构成
   - 二级资本构成
   - 风险加权资产明细
   - 资本充足率计算结果

2. **G02报表**：风险加权资产计算表
   - 信用风险加权资产
   - 市场风险加权资产
   - 操作风险加权资产
   - 风险加权资产合计

3. **G03报表**：资本构成表
   - 实收资本
   - 资本公积
   - 盈余公积
   - 未分配利润
   - 其他资本工具

4. **G04报表**：资本充足率汇总表
   - 资本充足率
   - 一级资本充足率
   - 核心一级资本充足率
   - 杠杆率

**数据来源**：
- 资产负债表
- 风险加权资产计算表
- 资本构成明细表
- 监管指标监控系统

**报送流程**：
1. **数据采集**：从业务系统采集相关数据
2. **数据校验**：校验数据准确性、完整性
3. **报表生成**：按照监管格式生成报表
4. **内部审核**：财务部、风险管理部、合规部审核
5. **领导审批**：分管领导、行长审批
6. **报送提交**：通过监管报送系统提交
7. **报送确认**：确认报送成功，保存报送记录

**关键要求**：
- 数据必须准确，不得有误
- 格式必须符合监管要求
- 必须在规定时间内报送
- 需要保存完整的审核记录

#### 场景二：资产质量报表（1104报表）

**业务背景**：
- 报送频率：**月度报送**
- 报送时间：每月结束后10个工作日内
- 监管部门：银保监会
- 报表编号：G11-G14系列

**报表内容**：
1. **G11报表**：贷款五级分类明细表
   - 正常类贷款
   - 关注类贷款
   - 次级类贷款
   - 可疑类贷款
   - 损失类贷款

2. **G12报表**：不良贷款明细表
   - 不良贷款余额
   - 不良贷款率
   - 不良贷款分布（按行业、地区、客户类型）

3. **G13报表**：贷款损失准备明细表
   - 一般准备
   - 专项准备
   - 特种准备
   - 拨备覆盖率

4. **G14报表**：资产质量汇总表
   - 不良贷款率
   - 拨备覆盖率
   - 贷款损失准备充足率

**数据来源**：
- 贷款分类系统
- 不良贷款管理系统
- 拨备计提系统
- 监管指标监控系统

**报送流程**：
与资本充足率报表类似，但审核重点不同：
- 重点关注不良贷款分类的准确性
- 关注拨备计提的充分性
- 关注资产质量变化趋势

#### 场景三：流动性报表（1104报表）

**业务背景**：
- 报送频率：**月度报送**
- 报送时间：每月结束后10个工作日内
- 监管部门：银保监会
- 报表编号：G21-G24系列

**报表内容**：
1. **G21报表**：流动性覆盖率计算表
   - 优质流动性资产明细
   - 未来30天现金净流出
   - 流动性覆盖率

2. **G22报表**：净稳定资金比例计算表
   - 可用稳定资金
   - 所需稳定资金
   - 净稳定资金比例

3. **G23报表**：存贷比计算表
   - 贷款余额
   - 存款余额
   - 存贷比

4. **G24报表**：流动性风险监测表
   - 流动性缺口
   - 流动性风险指标
   - 流动性风险预警

**数据来源**：
- 资产负债表
- 流动性风险监测系统
- 资金管理系统
- 监管指标监控系统

**报送流程**：
- 重点关注流动性指标的准确性
- 关注流动性风险的变化趋势
- 关注流动性应急预案的执行情况

### 2.2 专项监管报表报送

#### 场景四：大额风险暴露报表

**业务背景**：
- 报送频率：**季度报送**
- 报送时间：每季度结束后15个工作日内

   - 单一客户集中度
   - 大额客户明细

2. **G32报表**：单一行业大额风险暴露表
   - 单一行业授信总额
   - 单一行业集中度
   - 大额行业明细

3. **G33报表**：关联交易大额风险暴露表
   - 关联交易总额
   - 关联交易集中度
   - 关联交易明细

**数据来源**：

- 监管指标监控系统

**报送流程**：
- 重点关注集中度风险的准确性
- 关注大额风险暴露的变化
- 关注关联交易的合规性

#### 场景五：压力测试报表

**业务背景**：
- 报送频率：**年度报送**
- 报送时间：每年结束后3个月内
- 监管部门：银保监会
- 报表编号：G41-G43系列

**报表内容**：
1. **G41报表**：压力测试方案表
   - 压力情景设计
   - 测试方法
   - 测试范围

2. **G42报表**：压力测试结果表
   - 资本充足率压力测试结果
   - 流动性压力测试结果
   - 盈利压力测试结果

3. **G43报表**：压力测试分析报告
   - 测试结果分析
   - 风险识别
   - 应对措施

**数据来源**：
- 压力测试系统
- 风险模型系统
- 监管指标监控系统

**报送流程**：
- 重点关注压力测试的完整性
- 关注测试结果的准确性
- 关注应对措施的有效性

### 2.3 临时监管报表报送

#### 场景六：重大风险事件报告

**业务背景**：
- 报送频率：**事件触发**
- 报送时间：事件发生后24小时内
- 监管部门：银保监会
- 报告类型：重大风险事件报告

**报告内容**：
1. **事件基本情况**
   - 事件发生时间、地点
   - 事件性质、严重程度
   - 事件影响范围

2. **事件原因分析**
   - 直接原因
   - 根本原因
   - 相关因素

3. **事件影响评估**
   - 对业务的影响
   - 对财务状况的影响
   - 对声誉的影响

4. **应对措施**
   - 已采取的措施
   - 计划采取的措施
   - 应急预案执行情况

5. **整改措施**
   - 短期整改措施
   - 长期整改措施
   - 整改时间表

**数据来源**：
- 风险事件管理系统
- 应急预案管理系统
- 监管指标监控系统

**报送流程**：
1. **事件识别**：系统自动识别或人工发现重大风险事件
2. **事件评估**：评估事件的严重程度和影响范围
3. **报告编写**：编写重大风险事件报告
4. **紧急审核**：快速审核报告内容
5. **紧急报送**：24小时内报送监管部门
6. **跟踪反馈**：跟踪监管部门反馈，及时补充信息

**关键要求**：
- 必须在24小时内报送
- 报告内容必须真实、准确、完整
- 需要持续跟踪和更新

#### 场景七：监管检查配合

**业务背景**：
- 报送频率：**检查触发**
- 报送时间：根据监管检查要求
- 监管部门：银保监会及其派出机构
- 报告类型：监管检查配合材料

**报告内容**：
1. **检查配合材料**
   - 各类业务报表
   - 内部管理制度
   - 风险控制措施
   - 合规检查记录

2. **检查问题说明**
   - 问题描述
   - 问题原因
   - 整改措施
   - 整改时间表

3. **检查整改报告**
   - 整改措施执行情况
   - 整改效果评估
   - 后续改进计划

**数据来源**：
- 各类业务系统
- 合规管理系统
- 监管指标监控系统

**报送流程**：
1. **材料准备**：根据检查要求准备相关材料
2. **材料审核**：审核材料的完整性和准确性
3. **材料报送**：按时报送检查材料
4. **问题整改**：针对检查发现的问题进行整改
5. **整改报告**：提交整改报告

## 三、系统功能设计思路

### 3.1 报表模板管理

**功能描述**：
- 管理各类监管报表模板
- 支持模板的创建、编辑、删除
- 支持模板版本管理
- 支持模板的导入导出

**核心功能**：
1. **模板库管理**
   - 1104报表模板（资本充足率、资产质量、流动性等）
   - 专项报表模板（大额风险暴露、压力测试等）
   - 临时报表模板（重大风险事件报告等）
   - 自定义报表模板

2. **模板配置**
   - 报表基本信息（名称、编号、频率、时间要求等）
   - 报表结构（表头、表体、表尾等）
   - 数据字段映射（业务字段到报表字段的映射）
   - 计算公式配置
   - 校验规则配置

3. **模板版本管理**
   - 模板版本历史
   - 版本对比
   - 版本回滚

### 3.2 数据采集与校验

**功能描述**：
- 自动采集报表所需数据
- 数据校验和清洗
- 数据质量检查

**核心功能**：
1. **数据采集**

- 计算公式执行

**技术实现**：
- 使用模板引擎（Jinja2、Freemarker等）
- 支持Excel、PDF等格式模板

**功能**：
- 多数据源接入
- 数据采集任务调度
- 数据采集日志记录

**技术实现**：
- 使用ETL工具（Airflow、Kettle等）
- 支持数据库、API、文件等多种数据源
- 支持增量采集和全量采集

#### 模块三：数据校验引擎

**功能**：
- 数据完整性校验
- 数据准确性校验

- 规则引擎（Drools、Easy Rules等）

- **报表格式变化**：报表格式可能调整

- **报送延迟**：报送延迟可能面临监管处罚
- **数据泄露**：数据泄露可能面临监管处罚

**应对措施**：
- 严格的数据校验和审核
- 严格的时间管理
- 严格的数据安全控制
